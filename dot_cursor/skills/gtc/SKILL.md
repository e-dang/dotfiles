# gtc — Graphite track current branch

Run at **task start** to register the current branch with Graphite and set conventions for all work in this session.

Complete all steps below **before starting any implementation work**.

## Cursor context

Cursor automatically creates the worktree and branch from the base branch selected in the Agents UI. **Do not** set up git yourself:

- Do not run `git worktree add` or any other worktree commands
- Do not create a new branch (`git checkout -b`, `git branch`, etc.)

You may rename the current branch to match the naming convention below.

## Branch naming

All branches must be prefixed with `edang/`.

- **With a Jira ticket:** if the user provides a Jira link, extract the ticket ID and prefix the branch as `edang/<ticket-id>-<short-description>`.
  - Example: `https://envoycom.atlassian.net/browse/VIS-17354` → `edang/VIS-17354-calendar-preset-ranges`
- **Without a Jira ticket:** `edang/<short-description>`
  - Example: `edang/calendar-preset-ranges`

Keep the description after the prefix short, lowercase, and hyphen-separated.

If the current branch name does not match, rename it:

```bash
git branch -m edang/<ticket-id>-<short-description>
```

Then re-track:

```bash
gt track --force
```

## Track the branch

1. Link the current branch to its Graphite parent:

```bash
gt track --force
```

`--force` sets the parent to the most recent tracked ancestor, which matches a branch created from another stack branch.

If tracking fails or the parent is wrong:

```bash
gt log short --stack --no-interactive
gt track --parent <parent-branch>
gt track --force
```

The parent must already be tracked by Graphite (typically `main` or a branch lower in the stack).

2. Verify and show the stack to the user. Run:

```bash
gt log short --stack --no-interactive
```

**Paste the full output into the chat** so the user can verify the branch structure is correct. Do not begin implementation work until this output has been posted.

## Conventions for all work in this session

Apply these for the remainder of this task:

### Commits

When it comes time to commit you will not mention anything about AI generated code or how the commit messages are AI generated or what model you are, you will keep commit messages short but descriptive.

- No "Generated by", "Co-authored-by", model names, or AI disclaimers
- One line subject, imperative mood, under ~72 characters
- Optional short body only if the why isn't obvious

### Graphite stack context

This session runs in a Cursor-managed worktree and will be submitted as a stack of PRs.

- Do not use `gh pr create` — use Graphite (`gt ss`) when ready to submit
- Keep changes scoped to this branch
