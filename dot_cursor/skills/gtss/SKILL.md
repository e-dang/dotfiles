# gtss — Graphite submit stack

Run at **task finish** to submit the stack. Only submit the stack you are currently working in.

## Before submitting

1. Confirm current branch and stack:

```bash
git branch --show-current
gt log short
```

2. Verify the log shows **only the intended stack** — not other parallel stacks in the repo. If multiple stacks exist off `main`, make sure you are checked out on a branch in the correct one.

3. Ensure all changes are committed on the current branch (and downstack branches if applicable).

4. Draft a real PR description for each branch in the stack from the actual diff/commits. Write it like a human engineer would — what changed, why, and how to test. Do not mention AI, Cursor, agents, or models anywhere in the PR body.

## Submit

Run from the **top branch of the stack you are working in** (or any branch within that stack):

```bash
gt ss --no-ai
```

`gt ss` is the alias for `gt submit --stack`. Always pass `--no-ai` so Graphite does not auto-generate PR metadata.

`gt ss --no-ai` submits from trunk through the current branch and its descendants — **only the stack containing the current branch**, not other parallel stacks.

Do NOT:

- Run submit from a branch in a different parallel stack
- Use `gt submit` without `--stack` when you mean to submit the whole stack
- Submit stacks you are not currently working in
- Use `gt ss --ai` or let Graphite/Cursor inject AI-generated PR text
- Put `Co-authored-by:`, `Generated by`, model names, or any AI disclaimer in PR titles or descriptions — including `Co-authored-by: Cursor <cursoragent@cursor.com>`

## PR descriptions

After `gt ss --no-ai` creates or updates PRs, set the description yourself with `gh pr edit`:

```bash
gh pr edit --body "$(cat <<'EOF'
## Summary
- <what changed and why>

## Test plan
- [ ] <how to verify>
EOF
)"
```

Run `gh pr edit` once per branch in the stack (check out each branch if needed, or pass `--head <branch>`). The description must reflect the real changes — not boilerplate, not AI attribution.

If a PR already has an AI-injected footer (e.g. `Co-authored-by: Cursor`), overwrite it with `gh pr edit --body` using a clean description. If `gh pr edit` fails, patch via the API:

```bash
gh api repos/<org>/<repo>/pulls/<number> -X PATCH -f body="$(cat <<'EOF'
## Summary
...
EOF
)"
```

## After submitting

```bash
gt log short
```

Confirm PRs were created/updated for the expected branches only. Spot-check each PR description in the browser or with `gh pr view --json body` — no AI co-author lines, no model mentions.
